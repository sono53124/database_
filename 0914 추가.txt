제약조건 (컬럼 정의시 제약, 제한사항을 두는것)
목적 : 결점이 없는 데이터만 받아들이기 위해 ( 무결성 데이터를 위함)
종류
	Unique : 중복을 허용하지 않는 제약조건 (단 null이 허용되버린다
	not null: null을 허용하지 않는 제약조건
	Primary key: Unique + not null
	Default: 기본값 ex) regdate	date	default sysdata
			point	number	default 10000
	Check : 범위를 제한
	Foreign key (외래키)

정규화	중복되는 데이터는 별도의 테이블로 분리시키자
	얘도 테이블이므로, 반드시 primary key가 존재해야 함

	분리된 카테고리를 부모라하고
	분리된 카테고리의 번호를 가지게 된것을 자식이라 부름
	부모의 주민번호는 Foreign key (외래키)
	본래의 고유번호는 프라임키

	중복된 데이터는 분리시키자는 설계 원칙
	
	정규화 시키고 난 후, 물리적으로 분리된 테이블을 어떻게 하면 합쳐서
	보여줘야하나?

조인문(join)	
	물리적으로 분리된 테이블을 마치 하나의 테이블처럼 합쳐서 보여줄수있는
